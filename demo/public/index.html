Typeset

An HTML pre-proces­sor for web ty­pog­ra­phy. Typeset pro­vides ty­po­graphic fea­tures used tra­di­tion­ally in ﬁne print­ing which re­main un­avail­able to browser lay­out en­gines. Typeset’s pro­cess­ing brings the fol­low­ing to your web­pages:

  • Real hang­ing punc­tu­a­tion
  • Optical mar­gin align­ment
  • Soft hy­phen in­ser­tion
  • Small caps de­tec­tion
  • Punctuation sub­sti­tu­tion



Get the code on GitHub





How?

Typeset does not re­quire any client-side JavaScript and uses less than a kilo­byte of CSS. Processed HTML & CSS ren­ders cor­rectly in Internet Explorer 5 and with­out any CSS. Typeset is avail­able as a plu­gin for grunt and gulp or you can in­clude it man­u­ally.

npm install typeset

Usage

var typeset = require('typeset');
var html = "&lt;p&gt;"Hello," said the world!&lt;/p&gt;";

html = typeset(html);

Then tweak Typeset.css to match the met­rics of your font and in­clude it on your page.

Options

Typeset ac­cepts an op­tional sec­ond ar­gu­ment con­tain­ing con­ﬁg­u­ra­tion:

  • ig­nore <string>
    CSS se­lec­tor to which Typeset will not process. Example: 
    typeset(html, { ignore: '.skip' });
  • only <string>
    CSS se­lec­tor to only ap­ply Typeset to. Example: 
    typeset(html, {
    ignore: '.skip'
});
  • dis­able <array>
    List of type­set fea­tures to dis­able. The fol­low­ing fea­tures may be dis­abled:
    • quotes
    • hyphenate
    • ligatures
    • smallCapsr
    • punctuation
    • hangingPunctuation
    • spaces

CLI

npm install -g typeset

Compile a ﬁle and print it to std­out:

typeset-js input.html

To cre­ate an out­put ﬁle, just add a sec­ond ar­gu­ment:

typeset-js input.html output.html

Use the --ignore op­tion to ig­nore spe­ciﬁc CSS se­lec­tors:

typeset-js input.html output.html --ignore ".skip"

About

This pro­ject started as a col­lec­tion of li­braries used by Blot for type­set­ting blog posts. This would not be pos­si­ble with­out the work of Bram Stein and Dr. Drang. This page is set in Charter by Matthew Carter and Fira Mono by Erik Spiekermann.
